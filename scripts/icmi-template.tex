% Pandoc 3+ emits column widths like
%   p{(\linewidth - N\tabcolsep) * \real{0.1234}}
% where \real{X} must expand to the raw number X so the surrounding
% dimension arithmetic resolves to a length. Defining \real to wrap its
% argument in a \dimexpr (e.g. 0.1234pt) turns the expression into
% dim*dim, which silently fails and makes each p-column ~\linewidth
% wide; \resizebox then over-shrinks the table. Keep the simple
% identity definition pandoc expects.
\providecommand{\real}[1]{#1}
